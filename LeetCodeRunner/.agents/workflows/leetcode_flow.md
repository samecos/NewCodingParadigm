---
description: 基于数据探针的 TDD 测试驱动刷题流
---

当你接到一项具体的增量开发任务（即某道 LeetCode 题目）时，请严格按以下步骤执行：

1. **查探针**：搜索并阅读该题目在 `problems/{题目名}/problem.md` 中定义的数据探针 (`Requirement` 和 `Design`)。
2. **写测试**：阅读探针的约束，据此在 `problems/{题目名}/solution.test.ts` 中写好针对该约束的断言逻辑。
3. **嵌标识**：在测试函数和即将编写的业务函数上方，打上 `@probe_ref: [探针ID]` 的注释标签。
4. **跑红蓝**：运行 `npm run test -- problems/{题目名}/solution.test.ts`（此时应失败，呈现红色）。
5. **填逻辑**：在 `problems/{题目名}/solution.ts` 中编写纯函数，使其满足测试要求通过（呈现绿色）。如有必要，在此过程调用重构。
6. **验收总结**：检查是否使用了额外的状态或类，精简代码，并将通过的结果整理回反馈。
