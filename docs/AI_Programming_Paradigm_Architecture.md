# AI编程新范式：核心理论与架构设计

## 1. 概述

在传统的软件工程中，人类扮演着从需求分析到代码编写的执行者角色。而在“AI编程新范式”中，人类的定位实现了彻底的升维：从“代码搬砖者”转变为“规则制定者”、“架构规划者”与“产品验收者”。AI 则从单纯的代码提示工具，转变为贯穿项目全生命周期的核心协作者。

本架构设计的核心旨在解决**AI协作过程中的项目长时记忆与知识的准确传承**这一最大痛点，并提出基于“数据探针”和“项目级碎片化知识商店”的整体解决方案。

## 2. 需求与论证阶段：全维度尽调与评估

在项目立项与论证阶段，除了从上级或客户获取基础需求外，AI 将充分发挥其基于广域网的信息检索与结构化分析能力。

- **多维度评估**：AI 从需求、成果、经济效益、技术实现路径、投入产出比、社会价值等维度对项目进行全方位评估。
- **兜底验证**：作为项目落地的第一次兜底，AI 充当客观的“评估委员”，确保整个项目是否适合当前执行，为下一步计划的稳健推进提供数据支撑。

## 3. 实施阶段：机器友好型设计与测试前置

本范式下的软件工程实施阶段分为以下五个关键环节，其核心在于将人类共识转化为极其精确的“机器约束”。

### 3.1 定义阶段（控制性文档与数据探针）

- **范畴**：涵盖软件需求、概要设计、详细设计以及全流程的TDD（测试驱动开发）规划。
- **输出双向物料**：不仅产出人类易读的文档，更要产出带有**数据探针**的机器友好型设计文档。
- **数据探针机制**：通过在文案中埋入结构化数据（如特定标签、元数据注释、JSON Schema等），将知识颗粒度细化到“函数方法级别”。这为AI提供了精准的上下文导航，消除信息断层与幻觉。

### 3.2 测试与验证环境建设（测试集即基准）

- **基准固化**：针对第一阶段的文字性工作，以测试代码、测试数据集和验证程序的形式进行强制固化，形成不可逾越的“法律基准”。
- **探针延续与人类评审**：人类在此阶段的重心转为“规则评审”，并在测试代码中继续埋入数据探针，帮助AI理解验证逻辑和验收标准。

### 3.3 软件编写

- **环境约束**：根据既定的语言环境、数据仓库选型等进行代码的快速生成与编写。
- **覆盖率与健壮性**：不仅要求单元测试覆盖率达到 85% 以上，其核心目标更是提升测试代码和验证程序的健壮性，它们是检验 AI 产出代码质量的唯一客观准绳。

### 3.4 集成测试与功能验证

- 结合第2阶段搭建的基础设施，构建并运行自动化的集成测试、随机测试与端到端功能验证，确保系统级协同的可靠性。

### 3.5 交付验收

- 形成最终产品，并在高覆盖率的测试集和完备的机器约束下完成自动或半自动化验收。

## 4. 核心基础设施：项目级碎片化知识商店

为了解决长周期项目中上下文剧增与记忆丢失的痛点，新范式提出搭建一个“集 RAG 工作流与 MCP 技能于一体”的项目级碎片化知识商店。

### 4.1 核心组件

- **RAG (Retrieval-Augmented Generation) 记忆引擎**
  负责将项目的需求决策、架构设计、带有数据探针的约束碎片持久化为向量及关系型数据，形成项目的“长期跨度记忆”。能够做到精确到行/方法级别的内容溯源。
- **MCP (Model Context Protocol) 标准总线**
  提供标准化、松耦合的通信协议。在开发过程中，AI代理只需根据当前任务，通过MCP协议向知识商店“按需动态抽取”相关的函数级约束和领域知识，极大降低 LLM 每次请求的上下文负担。
- **Skills 业务流技能栈**
  将标准的开发规范动作和自动化流程（如“提取项目级探针”、“生成验证代码”、“对源代码与知识库进行一致性巡检”）沉淀为可复用的 AI 技能集合。

### 4.2 知识的传承与赋能

在新范式下，程序员的核心心智转移到了方案与项目的整体规划上；而 AI 侧则聚焦于协同产生的知识能够被准确、长效地传承和复用。项目级知识商店即是承载这类知识传承的物理载体，它让机器永远保持对项目全局最清晰的认知。
